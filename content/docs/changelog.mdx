## 2026-02-01

### New Feature: Tree TUI Fuzzy Filtering

Press `/` in `rei intention tree --tui` to enter filter mode:

- Type any characters to fuzzy-match against intention titles

### New Feature: Property Existence Filter

Filter intentions by whether they have a property set:

```bash

rei intention list --where "workflow:exists"
```

### Custom Property Enhancements

- **`rei custom-property relabel`** â€” Renamed from `rename` for clarity

### Automatic State Machine Initialization

State machine properties scoped to a category are now automatically initialized when a category is assigned to an entity. Use `init-entities` to retroactively initialize existing entities.

### New Feature: Intention Grouping

Group intention list output by custom property values:

```bash
rei intention list --group-by priority
rei intention list --group-by workflow
rei intention list --context work --group-by priority
rei intention list --where "effort>=3" --group-by priority
```

Intentions are organized into groups by their property value, with counts shown for each group. Group sorting order:

- **State machine properties**: sorted by `stateOrder` (ascending), then alphabetically
- **Numeric properties**: sorted descending (highest values first)
- **Text/enum properties**: sorted alphabetically ascending
- **"(none)" group**: always last (intentions without the property)

Grouping only works with `--format default` and can be combined with all other filters.

## 2026-01-15

### New Command: Categories

Categories provide a flexible way to classify and organize your actions, notes, and intentions. Unlike contexts (which group root intentions by life domain), categories form a hierarchical tree that can be applied to any entity.

**Core commands:**

- `rei category create` â€” Create a new category with optional parent and description
- `rei category list` â€” List categories (hierarchical by default, or flat)
- `rei category tree` â€” View category hierarchy as a tree
- `rei category show` â€” Show details of a specific category
- `rei category retire` â€” Soft-delete a category (preserved for historical data)
- `rei category reactivate` â€” Bring back a retired category
- `rei category link-focus` â€” Link a category to a focus area for time tracking

**Hierarchical slugs:**

Categories support parent-child relationships. The slug is automatically computed:

```bash
rei category create "Work" -d "Work activities"        # slug: work
rei category create "Meeting" -p work                   # slug: work/meeting
rei category create "1-on-1" -p work/meeting           # slug: work/meeting/1-on-1
```

**Category assignment:**

Assign categories to entities for classification:

- `rei action set-category` / `clear-category` â€” Categorize actions
- `rei note set-category` / `clear-category` â€” Categorize notes
- `rei intention set-category` / `clear-category` â€” Categorize intentions

Categories can be linked to focus areas to enable category-based time tracking and reporting.

Break habits support abstinence tracking:

- `rei habit start-abstinence` â€” Start tracking how long you've avoided the behavior
- `rei habit log-occurrence` â€” Log when the behavior occurred (resets tracking)

**Flexible schedules:**
- `--free-weekly <count>` â€” At least N times per week
- `--free-monthly <count>` â€” At least N times per month

- `--free-yearly <count>` â€” At least N times per year

**Cue-based:**
