## 2026-02-28

### New Guide: Cycles

New **[Cycles](/docs/guides/cycles)** guide covering focus cycles, daily adherence tracking, focus inheritance, and cycle lifecycle. Includes getting started steps, daily workflow, and tips.

### Focus Inheritance & Adherence Tracking

- **`rei intention set-focus`** — Assign a focus area to an intention. Focus propagates down the intention hierarchy: children inherit their parent's focus unless they override it.
- **`rei intention clear-focus`** — Remove the focus area from an intention, resuming inheritance from parent.
- **`rei cycle status`** and **`rei cycle show`** — Now display per-day adherence (action count, percentage, adherent indicator ✓/✗) for each daily focus selection, plus an overall adherence summary.
- **`rei today`** — Shows focus adherence for today when a daily focus is selected and intentions have focus areas assigned.

### Feature Enhancements

- **`rei link list --all --json`** — JSON output for link listings

## 2026-02-26 (2)

### Category-Qualified Templates

Templates now support **hierarchical names** with `/` as a separator, enabling category-specific templates with automatic fallback resolution.

- `rei template edit notes task` — Create a template for task-category notes
- `rei template edit notes task/meeting` — Create a nested sub-category template
- Fallback chain: `task/meeting` → `task` → `default`

When creating a note with `--category` but no `--template`, the category slug is automatically used as the template name:

```bash
rei note new --category task
# → automatically uses "task" template with fallback to "default"
```

## 2026-02-25

### Lifecycle Properties (Auto-Complete)

State machine properties can now be designated as **lifecycle properties**. When an intention's lifecycle property reaches a terminal state, the system schedules automatic completion after a configurable delay.

- `rei custom-property set-lifecycle workflow --days 3` — Designate lifecycle property with auto-complete delay
- `rei custom-property revoke-lifecycle workflow` — Remove lifecycle designation
- CLI shows notification when entering a terminal state: "This intention will auto-complete in 3 day(s)"
- If the intention moves back to a non-terminal state, the scheduled completion is cancelled

### Cycle Select-Focus Relative Dates

The `rei cycle select-focus --date` flag now supports relative date expressions in addition to `YYYY-MM-DD`:

- `rei cycle select-focus --date yesterday` — Set focus for yesterday
- `rei cycle select-focus -d "3 days ago"` — Set focus for 3 days ago

### Note Move Command

New **`rei note move`** command to reassign notes between entities:

- `rei note move -n note_01abc -i intention_01xyz` — Move a note to a different intention
- `rei note move` — Use FZF to select both note and target entity
- Supports moving to intentions, actions, outcomes, reflections, habits, and disruptions

### Note Open from Stdin

**`rei note open --stdin`** reads content from stdin instead of opening an editor, enabling programmatic note updates from scripts and agents:

- `echo "# Updated" | rei note open note_01abc --stdin`

### Subscription Workers

Added 15 missing subscription workers to the documentation, including `embed-propagation`, `link-extraction`, `category-read-model`, `disruption-read-model`, `custom-property-read-model`, `collection-read-model`, `terminal-state-completion`, and more.

## 2026-02-18

### First-Class Links with Canonical URL Deduplication

Links are now first-class entities with their own lifecycle. URLs are automatically normalized — tracking parameters stripped, domain-specific rules applied (YouTube, Amazon, Twitter/X, etc.), and redirect chains resolved. The same URL attached to multiple entities shares a single canonical identity.

**New subcommands:**
- `rei link show [<link-id>]` — View link details with all attachments
- `rei link detach [<link-id>]` — Detach a link from an anchor (link continues to exist)
- `rei link title [<link-id>] [-t <title>] [--clear]` — Set or clear display title
- `rei link edit [<link-id>] --url <url>` — Correct canonical URL (manual override)
- `rei link remove [<link-id>]` — Soft-delete a link (can be restored)
- `rei link restore [<link-id>]` — Restore a previously removed link

**Updated commands:**
- `rei link add` — Now uses canonical URL deduplication; auto-restores previously removed links
- `rei link list --all` — Show all links across all entities
- `rei link list --all --include-removed` — Include soft-deleted links

Outcomes can now be recorded on both **Active** and **Completed** intentions. This allows retrospective evidence to be added after an intention is finished:

- `rei outcome record -i <completed-intention> "Results discovered later"` — Record evidence after completion

Future intentions block operational changes (actions, blockers) but outcome recording is allowed on both Active and Completed states.

### New Feature: State Machine Required Property Rules

States can now require specific property values before allowing transitions. This enables checklist-gated workflows where prerequisites must be met before advancing:

```bash
# Require deploy-ready=true before entering production
rei custom-property add-state-rule workflow \
  --state production --require deploy-ready=true

# Remove a rule
rei custom-property remove-state-rule workflow \
  --state production --property deploy-ready
```

When adding a rule with a boolean value, if the referenced property doesn't exist, it is automatically created as a boolean property.

### New Feature: Task Completion Links

Boolean properties can be automatically set based on task completion in linked notes:

```bash
# Link note tasks to a boolean property
rei intention link-tasks -i <id> \
  --note <note-id> --property deploy-ready

# Remove the link
rei intention unlink-tasks -i <id> \
  --note <note-id> --property deploy-ready
```

When all tasks in the linked note are complete, the property is set to `true`. When any task is reopened, it reverts to `false`. Combined with state rules, this enables checklist-gated workflow transitions.

### Feature Enhancements

- **`rei intention show --full`** — Now displays category slug in intention details

### New Feature: Intention Grouping

Group intention list output by custom property values:

```bash
rei intention list --group-by priority
rei intention list --group-by workflow
rei intention list --context work --group-by priority
rei intention list --where "effort>=3" --group-by priority
```

Intentions are organized into groups by their property value, with counts shown for each group. Group sorting order:

- **State machine properties**: sorted by `stateOrder` (ascending), then alphabetically
- **Numeric properties**: sorted descending (highest values first)
- **Text/enum properties**: sorted alphabetically ascending
- **"(none)" group**: always last (intentions without the property)

Grouping only works with `--format default` and can be combined with all other filters.

## 2026-01-15

### New Command: Categories

Categories provide a flexible way to classify and organize your actions, notes, and intentions. Unlike contexts (which group root intentions by life domain), categories form a hierarchical tree that can be applied to any entity.

**Core commands:**

- `rei category create` — Create a new category with optional parent and description
- `rei category list` — List categories (hierarchical by default, or flat)
- `rei category tree` — View category hierarchy as a tree
- `rei category show` — Show details of a specific category
- `rei category retire` — Soft-delete a category (preserved for historical data)
- `rei category reactivate` — Bring back a retired category
- `rei category link-focus` — Link a category to a focus area for time tracking

**Hierarchical slugs:**

Categories support parent-child relationships. The slug is automatically computed:

```bash
rei category create "Work" -d "Work activities"        # slug: work
rei category create "Meeting" -p work                   # slug: work/meeting
rei category create "1-on-1" -p work/meeting           # slug: work/meeting/1-on-1
```

**Category assignment:**

Assign categories to entities for classification:

- `rei action set-category` / `clear-category` — Categorize actions
- `rei note set-category` / `clear-category` — Categorize notes
- `rei intention set-category` / `clear-category` — Categorize intentions

Categories can be linked to focus areas to enable category-based time tracking and reporting.
