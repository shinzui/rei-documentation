Attach and manage documents for entities. When documents are copied into the workspace (via `doc add` or `doc attach --copy`), they are automatically committed to the workspace git repository.

## Subcommands

### add

Copy an external file into `workspace/docs/` and attach it to an entity. The file is renamed using the generated DocId to avoid collisions (e.g., `doc_01h455vb4pex.pdf`).

```bash
rei doc add <file> [options]
```

**Arguments:**
- `<file>` — Path to the source file to copy (required)

**Options:**
- `-i, --intention <id>` — Attach to intention (optional if fzf is available)
- `-a, --action <id>` — Attach to action (optional)
- `-o, --outcome <id>` — Attach to outcome (optional)
- `-r, --reflection <id>` — Attach to reflection (optional)
- `-n, --note <id>` — Attach to note (optional)
- `--actor <actor>` — Actor performing this action (overrides global `--actor`)

> **FZF Support:** If no entity is specified and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from available entities.

**Examples:**

```bash
# Copy a file into workspace/docs and attach to an intention
rei doc add ~/Downloads/architecture.pdf -i intention_01h455vb4pex

# Let fzf pick the entity interactively
rei doc add ~/Documents/spec.md

# Add to a note
rei doc add ~/Downloads/diagram.png -n note_01h455vb4pex
```

### attach

Attach an existing file to an entity. By default, uses the file path as-is. Use `--copy` to copy the file into the workspace first.

```bash
rei doc attach <file> [options]
```

**Arguments:**
- `<file>` — Path to the file to attach (required)

**Options:**
- `-i, --intention <id>` — Attach to intention (optional if fzf is available)
- `-a, --action <id>` — Attach to action (optional)
- `-o, --outcome <id>` — Attach to outcome (optional)
- `-r, --reflection <id>` — Attach to reflection (optional)
- `-n, --note <id>` — Attach to note (optional)
- `--actor <actor>` — Actor performing this action (overrides global `--actor`)
- `--copy` — Copy the file into `workspace/docs/` before attaching (names file by DocId)

> **FZF Support:** If no entity is specified and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from available entities.

**Examples:**

```bash
# Attach a workspace-relative document to an intention
rei doc attach ./architecture.md -i intention_01h455vb4pex

# Let fzf pick the entity interactively
rei doc attach ./architecture.md

# Attach to an action
rei doc attach ./meeting-notes.md -i intention_01h455vb4pex -a action_01h455vb4pex

# Attach to a note
rei doc attach ./diagram.pdf -n note_01h455vb4pex

# Copy an external file into workspace/docs, then attach
rei doc attach ~/Downloads/report.pdf -i intention_01h455vb4pex --copy
```

### list

List attached documents. By default, docs whose anchor is an archived note are hidden from output; pass `--include-archived` to bring them back.

```bash
rei doc list [options]
```

**Options:**
- `-i, --intention <id>` — Filter by intention
- `-a, --action <id>` — Filter by action
- `-o, --outcome <id>` — Filter by outcome
- `-r, --reflection <id>` — Filter by reflection
- `-n, --note <id>` — Filter by note
- `--include-archived` — Include docs whose parent note is archived. Default behavior excludes them (they remain reachable by ID via `rei doc open`). Each row brought back by this flag is tagged `[archived parent]`.

**Examples:**

```bash
# List docs for an intention
rei doc list -i intention_01h455vb4pex

# List docs for a note
rei doc list -n note_01h455vb4pex
```

After archiving the note, the same listing is empty by default:

```text
$ rei doc list --note note_01h455vb4pex
No documents for note: note_01h455vb4pex

$ rei doc list --note note_01h455vb4pex --include-archived
Documents for note: note_01h455vb4pex
  doc_01abc... [archived parent]
    Path: workspace/docs/example.pdf
    Attached: 2026-04-29 14:00
```

`rei doc open <doc-id>` continues to work without any flag because `open` resolves by ID regardless of parent-note status.

### open

Open a document in your editor.

```bash
rei doc open [doc-id]
```

**Arguments:**
- `[doc-id]` — Document ID (optional if fzf is available)

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from available documents.

**Example:**

```bash
rei doc open doc_01h455vb4pex
```

### remove

Remove a document attachment from an entity.

```bash
rei doc remove [doc-id]
```

**Arguments:**
- `[doc-id]` — Document ID (optional if fzf is available)

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from available documents.

**Examples:**

```bash
# Remove a specific document
rei doc remove doc_01h455vb4pex

# Let fzf pick the document
rei doc remove
```
