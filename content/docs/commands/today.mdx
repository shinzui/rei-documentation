Display a daily dashboard showing today's summary. This is your primary view for understanding what needs attention each day.

```bash
rei today [options]
```

**Options:**
- `-c, --context <context>` — Filter by context (e.g., `work`, `personal`)
- `--include-dormant` — Include dormant intentions (excluded by default)
- `--tui` — Launch interactive TUI dashboard

**Section filters** (combinable — when any is provided, only selected sections are shown):
- `--actions` — Show only today's actions
- `--reminders` — Show only triggered reminders
- `--intentions` — Show only intentions, completed today, tasks, blockers, and related sections
- `--habits` — Show only habits (scheduled, flexible, and cue-based)
- `--panels` — Show only dashboard panels

## Dashboard Sections

The dashboard displays the following sections:

### Active Cycle & Focus

If you have an active focus cycle, shows:
- Cycle name and progress (e.g., "Day 3/14, 11 days remaining")
- Today's designated focus area (if set)
- Focus adherence for today (e.g., "Focus Adherence: 75% (6/8 actions) ✓ Adherent") when a daily focus is selected and intentions have focus areas assigned

### Triggered Reminders

Reminders that have triggered and need attention, shown with their scheduled time and anchor type.

### Active Intentions

Your active intentions, either:
- **Grouped by context** (when no filter applied) — organized under context headers like "▸ work", "▸ personal"
- **Flat list** (when filtered by `-c`) — showing only matching intentions

Each intention shows:
- Active days count (e.g., `[45d]`) — the number of distinct days with recorded actions
- Action count if you've taken actions today
- "(blocked)" indicator if it has unresolved blockers
- Deadline date if set (e.g., `due: 2026-02-15`)

### Completed Today

Intentions completed today, grouped by context (or flat when filtered).

### Tasks

Tasks organized by urgency:
- **Overdue** — Tasks past their due date (shown in red)
- **Due Today** — Tasks due today (shown in yellow)
- **Upcoming** — Tasks due within the next 7 days

### Today's Actions

Chronological list of actions recorded today, showing:
- Time performed (in your local timezone)
- Description
- Duration (if tracked)
- Attached notes (if any)

### Blockers

Active blockers with age indicator (e.g., "3 days").

### Habits

Scheduled habits for today with completion status:
- ✓ indicates completed
- ○ indicates pending
- Active days indicator (e.g., `[30d]`) showing distinct days with recorded actions

### Flexible Habits

Habits with flexible schedules (FreeWeekly, FreeMonthly, FreeQuarterly, FreeYearly) showing:
- Progress indicator (e.g., "2/3 this week", "1/5 this month")
- Days performed in the current period (e.g., "Mon, Wed")
- Color-coded urgency:
  - Green: On track to meet target
  - Yellow: Needs attention to meet target
  - Red: Target unachievable with days remaining

### Cue-Based Habits

Habits triggered by context/cues rather than schedules, showing:
- Cue trigger description
- Target frequency (e.g., "2/week")
- When last performed (e.g., "today", "yesterday", "3 days ago")
- Active days indicator (e.g., `[30d]`) showing distinct days with recorded actions

### Pending Reflections

Scheduled reflections that are due but not yet completed.

## Examples

```bash
# Show full dashboard
rei today

# Filter to work context only
rei today --context work

# Show only today's actions
rei today --actions

# Show only habits
rei today --habits

# Show actions and reminders together
rei today --actions --reminders

# Combine with context filter
rei today --context work --intentions

# Include dormant intentions
rei today --include-dormant

# Launch interactive TUI mode
rei today --tui
```

## TUI Mode

The `--tui` flag launches an interactive terminal UI where you can:
- Navigate between sections with `j`/`k` or arrow keys
- Press `Enter` on an intention to open a full detail view (same as `intention tree --tui` — see [intention docs](/docs/commands/intention) for detail view contents)
- Press `o` to open notes from the detail view
- Press `Esc` to close detail view, `q` to quit

**Note:** TUI mode requires a terminal that supports ANSI escape codes.

## Dashboard Panels

You can define custom intention query panels that appear at the end of the today output. Panels are configured in your `rei.yaml` config file under `dashboard.panels`.

```yaml
dashboard:
  panels:
    - label: "Kanban Board"
      where: "workflow:exists"
      group_by: "workflow"

    - label: "In Development"
      where: "version-phase=development"
      context: "work"

    - label: "Queue"
      state_tag: "queue"
      category: "projects"
```

Each panel supports:
- `label` (required) — Display name shown as the panel header
- `where` — Property filter expression (same syntax as `intention list --where`)
- `state_tag` — Filter by state machine system tag (`queue`, `active`, `done`, `terminal`)
- `group_by` — Property key to group results by
- `context` — Limit to intentions within this context
- `category` — Limit to intentions within this category slug

All options combine with AND logic. Panels with no matching intentions show "No matching intentions".

For full details, see the [Dashboard Panels Guide](/docs/guides/dashboard).

## Notes

- Times are displayed in your local timezone
- The dashboard requires a database backend (PostgreSQL)
- Context filtering shows intentions whose root ancestor has the matching context
- Linked habits inherit context from their linked intention
- Dormant intentions (inactive for 30+ days) are excluded by default; use `--include-dormant` to show them
