Manage blockers for intentions and habits. Blockers represent obstacles or things preventing progress. Intention blockers can also be classified with a category and extended with user-defined custom properties; see [`rei custom-property`](/docs/commands/custom-property) for how to define and assign properties. These commands do not currently apply to habit blockers.

## Subcommands

### declare

Declare a new blocker for an intention or habit.

```bash
rei blocker declare [description] [options]
```

**Arguments:**
- `[description]` — Description of the blocker (optional, can be provided via stdin)

**Options:**
- `-i, --intention <id>` — Intention ID (optional if fzf is available)
- `--habit <id>` — Habit ID (for habit blockers instead of intention blockers)
- `--actor <actor>` — Actor performing this action (overrides global `--actor`)

> **FZF Support:** If neither `-i` nor `--habit` is provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active intentions.

**Examples:**

```bash
# Declare a blocker on an intention with explicit ID
rei blocker declare -i intention_01h455vb4pex "Waiting for API credentials"

# Let fzf pick the intention interactively
rei blocker declare "Waiting for API credentials"

# Declare a blocker on a habit (auto-pauses the habit)
rei blocker declare --habit habit_01abc... "Gym is closed for renovation"

# Declare via stdin
echo "Need to learn GraphQL before proceeding" | rei blocker declare -i intention_01h455vb4pex
```

**Notes:**
- Habit blockers automatically pause the habit when declared
- Cannot specify both `--intention` and `--habit` at the same time

### resolve

Resolve (remove) a blocker.

```bash
rei blocker resolve [blocker-id] [options]
```

**Arguments:**
- `[blocker-id]` — Blocker ID (optional if fzf is available)

**Options:**
- `-i, --intention <id>` — Intention ID (optional if fzf is available)
- `--habit <id>` — Habit ID (for habit blockers instead of intention blockers)
- `--actor <actor>` — Actor performing this action (overrides global `--actor`)

> **FZF Support:** If IDs are not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from intentions/habits with blockers, then select the blocker.

**Examples:**

```bash
# Resolve intention blocker with explicit IDs
rei blocker resolve blocker_01h455vb4pex -i intention_01h455vb4pex

# Let fzf pick the intention and blocker
rei blocker resolve

# Resolve a habit blocker (auto-resumes habit if last blocker)
rei blocker resolve --habit habit_01abc... blocker_01xyz...

# Let fzf pick habit with blockers, then pick blocker
rei blocker resolve --habit
```

**Notes:**
- When the last blocker on a habit is resolved, the habit automatically resumes
- Cannot specify both `--intention` and `--habit` at the same time

### list

List all active blockers.

```bash
rei blocker list [options]
```

**Options:**
- `--intentions` — Show only intention blockers
- `--habits` — Show only habit blockers

**Examples:**

```bash
# List all active blockers (intentions and habits)
rei blocker list

# List only intention blockers
rei blocker list --intentions

# List only habit blockers
rei blocker list --habits
```

**Output:**

When showing all blockers:
```
=== Intention Blockers ===
  • blocker_01abc...
    Intention: Learn GraphQL
    Waiting for course access

=== Habit Blockers ===
  • blocker_01xyz...
    Habit: Morning Gym
    Gym closed for renovation
```

### set-category

Assign a category to an intention blocker. Categories help classify and organize blockers for reporting and filtering, and they can auto-initialize category-scoped state machine properties on the blocker.

```bash
rei blocker set-category [-b BLOCKER_ID] CATEGORY_SLUG
```

**Arguments:**
- `CATEGORY_SLUG` — Category slug (e.g., 'work', 'personal/health')

**Options:**
- `-b, --blocker <id>` — Blocker ID (optional if fzf is available)

> **FZF Support:** If `-b` is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active intention blockers.

**Examples:**

```bash
# Set category with explicit blocker ID
rei blocker set-category -b blocker_01h455vb4pex external/dependency

# Let fzf pick the blocker
rei blocker set-category external/dependency
```

**Notes:**
- Applies to intention blockers only; habit blockers are not supported by this command.
- Any state machine custom property scoped to the category is auto-initialized on the blocker when the category is assigned.

### set-property

Set a custom property on an intention blocker.

```bash
rei blocker set-property [-b BLOCKER_ID] PROPERTY_KEY VALUE [--force]
```

**Arguments:**
- `PROPERTY_KEY` — The key of the custom property to set
- `VALUE` — The value to assign (validated against property definition)

**Options:**
- `-b, --blocker <id>` — Blocker ID (optional if fzf is available)
- `--force` — Bypass state machine transition rules (still blocked from terminal states)

> **FZF Support:** If `-b` is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active intention blockers.

**Examples:**

```bash
# Set property with explicit blocker ID
rei blocker set-property -b blocker_01h455vb4pex severity high

# Let fzf pick the blocker
rei blocker set-property severity high

# Set a state machine property
rei blocker set-property -b blocker_01h455vb4pex triage-status investigating

# Skip intermediate states on a state machine property
rei blocker set-property -b blocker_01h455vb4pex triage-status resolved --force
```

**Notes:**
- Applies to intention blockers only; habit blockers are not supported by this command.
- The property must be defined first using `rei custom-property create`
- The property must be configured to apply to blockers (`--entity-type blocker`)
- Values are validated against the property definition (enum options, numeric ranges, etc.)
- `--force` only affects state machine properties — it bypasses transition rules but cannot leave terminal states.

## Habit Blocker Behavior

Habit blockers have automatic pause/resume behavior:

1. **Auto-pause:** When a blocker is declared on an active habit, the habit is automatically paused
2. **Auto-resume:** When the last blocker is resolved, the habit automatically resumes

This means you don't need to manually pause/resume habits when blockers are involved.

> **Note:** The `set-category`, `clear-category`, `set-property`, and `clear-property` subcommands above apply to intention blockers only. They do not act on habit blockers.

## Viewing Blocked Intentions

See all intentions with blockers:

```bash
rei intention blocked
```

This helps you identify what's stuck and why.

## Related Commands

- `rei intention blocked` — List all intentions with unresolved blockers
- `rei intention show <id>` — Shows blockers for a specific intention
- `rei habit show <id>` — Shows blockers for a specific habit
- `rei habit pause` — Manually pause a habit (without a blocker)
- `rei habit resume` — Manually resume a paused habit
