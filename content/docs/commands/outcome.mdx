Outcomes record observable evidence that something changed. Unlike actions (which track effort), outcomes track results. Outcomes can be recorded on both **Active** and **Completed** intentions, allowing retrospective evidence to be added after an intention is finished. Outcomes can also be classified with a category and extended with user-defined custom properties in the same way as intentions; see [`rei custom-property`](/docs/commands/custom-property) for how to define and assign properties.

## Subcommands

### record

Record an outcome for an intention.

```bash
rei outcome record [description] [options]
```

**Arguments:**
- `[description]` — Description of the outcome (optional, can be provided via stdin)

**Options:**
- `-i, --intention <id>` — Intention ID (optional if fzf is available)
- `-t, --at <time>` — When the outcome was achieved (optional, defaults to now)
- `--actor <actor>` — Actor performing this action (overrides global `--actor`)

> **FZF Support:** If `-i` is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active intentions.

> **Completed Intentions:** You can record outcomes on completed intentions too. This is useful when results or evidence are discovered after an intention has been marked complete.

**Relative Time:**

The `--at` option accepts human-readable relative time expressions:
- Keywords: `now`, `today`, `yesterday`, `tomorrow`
- Natural expressions: `2 hours ago`, `30 minutes ago`, `1 day ago`
- Short forms: `-2h`, `-30m`, `-1d`, `2h`, `30m`
- ISO 8601: `2024-01-15T14:30:00Z`, `2024-01-15T14:30:00+09:00`

**Examples:**

```bash
# Record an outcome with explicit intention
rei outcome record -i intention_01h455vb4pex "Deployed to production successfully"

# Let fzf pick the intention interactively
rei outcome record "Deployed to production successfully"

# Record via stdin
echo "All tests passing, 95% coverage" | rei outcome record -i intention_01h455vb4pex

# Record an outcome that happened 2 hours ago
rei outcome record "Feature approved by client" --at "2 hours ago"

# Record yesterday's outcome
rei outcome record -i intention_01abc... "Milestone completed" -t yesterday
```

### set-category

Assign a category to an outcome. Categories help classify and organize outcomes for reporting and filtering, and they can auto-initialize category-scoped state machine properties on the outcome.

```bash
rei outcome set-category OUTCOME_ID CATEGORY_SLUG
```

**Arguments:**
- `OUTCOME_ID` — Outcome ID to categorize
- `CATEGORY_SLUG` — Category slug (e.g., 'work', 'personal/health')

**Examples:**

```bash
# Set category on an outcome
rei outcome set-category outcome_01h455vb4pex work/milestone

# Use a nested category slug
rei outcome set-category outcome_01h455vb4pex personal/health
```

**Notes:**
- Any state machine custom property scoped to the category is auto-initialized on the outcome when the category is assigned.

### clear-category

Remove the category from an outcome.

```bash
rei outcome clear-category OUTCOME_ID
```

**Arguments:**
- `OUTCOME_ID` — Outcome ID

**Examples:**

```bash
# Clear the category from an outcome
rei outcome clear-category outcome_01h455vb4pex
```

### set-property

Set a custom property on an outcome.

```bash
rei outcome set-property OUTCOME_ID KEY VALUE [--force]
```

**Arguments:**
- `OUTCOME_ID` — Outcome ID
- `KEY` — Property key
- `VALUE` — Property value (validated against property definition)

**Options:**
- `--force` — Bypass state machine transition rules (still blocked from terminal states)

**Examples:**

```bash
# Set a property on an outcome
rei outcome set-property outcome_01h455vb4pex impact high

# Set a numeric property
rei outcome set-property outcome_01h455vb4pex confidence 4

# Skip intermediate states on a state machine property
rei outcome set-property outcome_01h455vb4pex review-state accepted --force
```

**Notes:**
- The property must be defined first using `rei custom-property create`
- The property must be configured to apply to outcomes (`--entity-type outcome`)
- Values are validated against the property definition (enum options, numeric ranges, etc.)
- `--force` only affects state machine properties — it bypasses transition rules but cannot leave terminal states.

### clear-property

Remove a custom property from an outcome.

```bash
rei outcome clear-property OUTCOME_ID KEY
```

**Arguments:**
- `OUTCOME_ID` — Outcome ID
- `KEY` — Property key to clear

**Examples:**

```bash
# Clear a property from an outcome
rei outcome clear-property outcome_01h455vb4pex impact
```

## Actions vs Outcomes

| Actions | Outcomes |
|---------|----------|
| What you did | What changed |
| Effort | Results |
| "Wrote for 30 minutes" | "Published blog post" |
| Process | Evidence |

Both are valuable:
- Actions track your investment of time and energy
- Outcomes track the results of that investment

## Viewing Outcomes

Outcomes appear in:
- `rei intention show --full` — When viewing an intention
- `rei intention timeline` — Chronological history
- Weekly reviews
