Categories provide a flexible way to classify and organize actions, notes, and intentions. Categories form a hierarchical tree structure and can be linked to focus areas for time tracking.

## Overview

Categories are created with a name and optional parent. The system automatically generates a slug (e.g., `work/meeting` for a "meeting" category under "work").

```bash
# Create a top-level category
rei category create "Work" -d "Work-related activities"

# Create a child category
rei category create "Meeting" -p work -d "Work meetings"

# List categories as a tree
rei category tree
```

## Subcommands

### create

Create a new category.

```bash
rei category create NAME [options]
```

**Arguments:**
- `NAME` — Name of the category (e.g., "Meeting", "Reading", "Coding")

**Options:**
- `-p, --parent PARENT_SLUG` — Parent category slug (creates hierarchical categories)
- `-d, --description DESC` — Description of the category
- `-f, --focus FOCUS_ID` — Link to a focus area for time tracking

**Examples:**

```bash
# Create a top-level category
rei category create "Work" -d "Work activities"

# Create a child category under Work
rei category create "Meeting" -p work

# Create with focus area link
rei category create "Deep Work" -f focus_01abc123...
```

**Output:**
```
Created category: work/meeting
  ID: category_01abc123...
  Name: Meeting
```

### list

List categories.

```bash
rei category list [options]
```

**Options:**
- `-a, --all` — Show all categories including retired
- `-d, --descriptions` — Show category descriptions
- `--flat` — Show flat list instead of hierarchy

**Examples:**

```bash
# List active categories (hierarchical by default)
rei category list

# List all including retired
rei category list --all

# Show with descriptions
rei category list --descriptions

# Flat list with descriptions
rei category list --flat --descriptions
```

**Output:**
```
Categories (4):

[*] Personal
  [*] Health
[*] Work
  [*] Meeting
```

With `--descriptions`:
```
Categories (4):

[*] Personal - Personal life activities
  [*] Health - Health and fitness
[*] Work - Work activities
  [*] Meeting - Work meetings
```

Status icons:
- `[*]` — Active category
- `[x]` — Retired category

### show

Show details of a category.

```bash
rei category show [SLUG]
```

**Arguments:**
- `[SLUG]` — Category slug (optional if fzf is available)

> **FZF Support:** If the slug is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from existing categories with a preview pane showing category details.

**Examples:**

```bash
# Show with explicit slug
rei category show work/meeting

# Let fzf pick the category
rei category show
```

**Output:**
```
Category: work/meeting
  ID: category_01abc123...
  Name: Meeting
  Description: Work meetings
  Usage: (usage text if set)
  Parent: category_01parent...
  Status: Active
  Note Guidance: .rei/category-guidance/work-meeting.md
  Created: 2026-01-15 10:30
```

If the category has note guidance configured, the output shows the path to the guidance file.

### tree

Show category hierarchy as a tree.

```bash
rei category tree
```

**Output:**
```
Categories (4):

[*] Personal
  [*] Health
[*] Work
  [*] Meeting
```

### retire

Retire a category. Retired categories are hidden from default listings but preserved for historical data.

```bash
rei category retire SLUG
```

**Example:**

```bash
rei category retire work/old-project
```

### reactivate

Reactivate a retired category.

```bash
rei category reactivate SLUG
```

**Example:**

```bash
rei category reactivate work/old-project
```

### set-description

Set or update a category's description. The description is shown in category listings and details.

```bash
rei category set-description SLUG [DESCRIPTION] [options]
```

**Arguments:**
- `SLUG` — Category slug
- `[DESCRIPTION]` — Description text (optional, see input methods below)

**Options:**
- `--stdin` — Read description from stdin
- `--clear` — Clear the description

**Input Methods:**
1. Direct argument: `rei category set-description work/meeting "Work meetings"`
2. From stdin: `echo "Work meetings" | rei category set-description work/meeting --stdin`
3. Open editor: `rei category set-description work/meeting` (opens $EDITOR if no argument)

**Examples:**

```bash
# Set description directly
rei category set-description work/meeting "Team meetings and syncs"

# Set description from stdin
echo "Personal health activities" | rei category set-description personal/health --stdin

# Clear description
rei category set-description work/meeting --clear

# Open editor to edit description
rei category set-description work/meeting
```

### link-focus

Link a category to a focus area, or unlink by omitting the focus ID.

```bash
rei category link-focus SLUG [FOCUS_ID]
```

**Arguments:**
- `SLUG` — Category slug
- `FOCUS_ID` — Focus area ID (omit to unlink)

**Examples:**

```bash
# Link to a focus area
rei category link-focus work/meeting focus_01abc123...

# Unlink from focus area
rei category link-focus work/meeting
```

### set-usage

Set usage text for a category. Usage text provides detailed guidance on when and how to use a category, helping disambiguate similar categories.

```bash
rei category set-usage SLUG [options]
```

**Arguments:**
- `SLUG` — Category slug

**Options:**
- `--stdin` — Read usage text from stdin (otherwise opens $EDITOR)

Unlike the short `description` (shown in lists), `usage` provides longer explanatory text for non-obvious categories.

**Examples:**

```bash
# Set usage via editor
rei category set-usage work/meeting

# Set usage from stdin
echo "Use for scheduled meetings with 2+ people. For 1-on-1s, use work/1-on-1 instead." | rei category set-usage work/meeting --stdin
```

### usage

Print the usage text for a category to stdout.

```bash
rei category usage SLUG
```

**Arguments:**
- `SLUG` — Category slug

**Examples:**

```bash
# Print usage to terminal
rei category usage work/meeting

# Pipe to clipboard
rei category usage work/meeting | pbcopy
```

### set-note-guidance

Set guidance text for notes in this category. This guidance is automatically included when using `rei note get-help` on notes assigned to this category.

```bash
rei category set-note-guidance SLUG [options]
```

**Arguments:**
- `SLUG` — Category slug

**Options:**
- `-c, --content <text>` — Guidance content (if not provided, opens editor)
- `--stdin` — Read content from stdin

Category guidance provides domain knowledge that applies to all notes in the category (e.g., "how to study kanji" for a `learning/japanese` category). It composes with intention-level guidance when both are present.

**Examples:**

```bash
# Set guidance via editor
rei category set-note-guidance learning/japanese

# Set guidance inline
rei category set-note-guidance learning/japanese -c "Focus on stroke order and radicals"

# Set guidance from file
cat guidance.md | rei category set-note-guidance learning/japanese --stdin
```

### clear-note-guidance

Remove guidance from a category.

```bash
rei category clear-note-guidance SLUG
```

**Arguments:**
- `SLUG` — Category slug

**Example:**

```bash
rei category clear-note-guidance learning/japanese
```

### print-note-guidance

Print the note guidance for a category to stdout. Useful for piping to other tools or reviewing guidance content.

```bash
rei category print-note-guidance SLUG
```

**Arguments:**
- `SLUG` — Category slug

**Examples:**

```bash
# Print guidance to terminal
rei category print-note-guidance learning/japanese

# Pipe to clipboard
rei category print-note-guidance learning/japanese | pbcopy
```

### improve-note-guidance

Improve note guidance for a category using AI. This invokes Claude to suggest improvements to the existing guidance.

```bash
rei category improve-note-guidance SLUG [INSTRUCTIONS] [options]
```

**Arguments:**
- `SLUG` — Category slug
- `[INSTRUCTIONS]` — Improvement instructions (optional, e.g., "add examples for stroke order")

**Options:**
- `--stdin` — Read improvement instructions from stdin
- `--debug` — Show the prompt without calling the agent

**Behavior:**
1. Loads current guidance for the category
2. Sends guidance and instructions to Claude
3. Opens editor with suggested improvements
4. On save, updates the category guidance

**Examples:**

```bash
# Improve with inline instructions
rei category improve-note-guidance learning/japanese "add examples for radicals"

# Improve via stdin
echo "focus on memorization techniques" | rei category improve-note-guidance learning/japanese --stdin

# Interactive improvement (opens editor for instructions)
rei category improve-note-guidance learning/japanese

# Debug: see the prompt without calling the agent
rei category improve-note-guidance learning/japanese "add more detail" --debug
```

## Category Assignment

Categories can be assigned to actions, notes, and intentions.

### Assign to Action

```bash
rei action set-category [-a ACTION_ID] CATEGORY_SLUG
rei action clear-category [-a ACTION_ID]
```

Uses FZF for action selection if `-a` not provided.

**Example:**

```bash
rei action set-category -a action_01abc... work/meeting
rei action clear-category -a action_01abc...
```

### Assign to Note

```bash
rei note set-category [-n NOTE_ID] CATEGORY_SLUG
rei note clear-category [-n NOTE_ID]
```

Uses FZF for note selection if `-n` not provided.

**Example:**

```bash
rei note set-category -n note_01abc... personal/health
rei note clear-category -n note_01abc...
```

### Assign to Intention

```bash
rei intention set-category [-i INTENTION_ID] CATEGORY_SLUG
rei intention clear-category [INTENTION_ID]
```

Uses FZF for intention selection if ID not provided.

**Example:**

```bash
rei intention set-category -i intention_01abc... work
rei intention clear-category intention_01abc...
```

## Hierarchical Slugs

When you create a category with a parent, the slug is automatically computed:

| Name | Parent | Resulting Slug |
|------|--------|----------------|
| Work | (none) | `work` |
| Meeting | work | `work/meeting` |
| 1-on-1 | work/meeting | `work/meeting/1-on-1` |

## Focus Area Integration

Categories can be linked to focus areas to enable:
- Automatic time allocation tracking
- Category-based reporting
- Time budgeting by activity type

```bash
# Create category linked to focus
rei category create "Deep Work" -f focus_01abc...

# Or link existing category
rei category link-focus deep-work focus_01abc...
```

## Tips

- **Start simple**: Begin with a few top-level categories, add children as needed
- **Use descriptive names**: Category names become slugs (lowercased, spaces to hyphens)
- **Retire instead of delete**: Categories are soft-deleted to preserve historical data
- **Link to focus areas**: Connect categories to focus areas for time tracking integration
