Manage daily focus cycles. A cycle is a bounded time period for tracking daily focus selections.

## Subcommands

### start

Start a new cycle.

```bash
rei cycle start [options]
```

**Options:**
- `-l, --length <days>` — Cycle length in days (required)
- `-n, --name <name>` — Optional name for the cycle
- `-d, --start-date <date>` — Start date (defaults to today, format: YYYY-MM-DD)
- `-t, --at <time>` — When the cycle started (relative time expression, e.g., "yesterday", "3 days ago")

**Examples:**

```bash
# Start a 10-day cycle
rei cycle start --length 10

# Start a named 7-day cycle
rei cycle start --length 7 --name "Week 1"

# Start a cycle with a specific start date
rei cycle start --length 10 --start-date 2025-01-01

# Backdate a cycle start (e.g., forgot to start yesterday)
rei cycle start --length 10 --at yesterday
```

**Notes:**
- Only one cycle can be active at a time
- Starting a new cycle when one is active will fail
- Use `--at` to backdate cycle starts (useful when you forgot to start on time)

### status

Show the status of the active cycle.

```bash
rei cycle status
```

**Output:**

```
Active Cycle:
  ID: cycle_01abc...
  Start: 2025-01-01
  End: 2025-01-10
  Length: 10 days
  Progress: 3/10 days with focus selected

Daily Focus Selections:
  2025-01-01: Connect   5 actions  80% ✓
  2025-01-02: Study     3 actions  67% ✓
  2025-01-03: Create    0 actions   —

  Overall: 74% avg (2/2 adherent days)
```

Each daily focus row shows:
- **Action count** — eligible actions recorded that day (excluding disruption-linked)
- **Adherence %** — percentage of actions whose intention (or inherited) focus matches the daily focus
- **Indicator** — ✓ if adherent (≥60%), ✗ if not
- Days with zero actions show `—` (no signal, not failure)

The overall summary shows average adherence and count of adherent days (only counting days with at least one action).

### select-focus

Select a focus for today (or change a previous day's selection).

```bash
rei cycle select-focus [focus-id] [options]
```

**Arguments:**
- `[focus-id]` — Focus ID (optional, interactive selection if omitted)

**Options:**
- `-d, --date <date>` — Date to set focus for (defaults to today). Accepts `YYYY-MM-DD`, relative expressions like `yesterday` or `3 days ago`, and other time expressions.

**Examples:**

```bash
# Interactive selection for today
rei cycle select-focus

# Direct selection for today
rei cycle select-focus focus_01abc...

# Set focus for a past date (absolute)
rei cycle select-focus focus_01abc... --date 2025-01-01

# Set focus for yesterday
rei cycle select-focus --date yesterday

# Set focus for 3 days ago
rei cycle select-focus -d "3 days ago"
```

**Notes:**
- Can only select focus for dates within the cycle bounds
- Can only select focus for today or past dates (no future planning)
- Selecting a new focus for a date that already has one will change it

### complete

Mark the active cycle as completed.

```bash
rei cycle complete [options]
```

**Options:**
- `-t, --at <time>` — When the cycle was completed (relative time expression, e.g., "yesterday", "2 hours ago")

**Examples:**

```bash
# Complete the active cycle now
rei cycle complete

# Backdate completion (e.g., forgot to complete yesterday)
rei cycle complete --at yesterday
```

**Notes:**
- Completed cycles are terminal — they cannot be modified
- You can complete a cycle before all days have focus selections
- Use `--at` to backdate cycle completions (useful when you forgot to mark completion)

### abandon

Abandon the active cycle.

```bash
rei cycle abandon
```

**Notes:**
- Abandoned cycles are terminal — they cannot be modified
- Use this when you want to start fresh without completing the current cycle

### extend

Extend the active cycle by additional days.

```bash
rei cycle extend <days>
```

**Arguments:**
- `<days>` — Number of days to add to the cycle (required, must be positive)

**Examples:**

```bash
# Extend by 3 days
rei cycle extend 3

# Extend by a week
rei cycle extend 7
```

**Notes:**
- Only works on active cycles
- Cannot extend completed or abandoned cycles
- The extension is added to the current cycle length

### reschedule

Adjust the start date and optionally the length of the active cycle. Useful when a cycle was created with the wrong start date.

```bash
rei cycle reschedule --start <date> [--length <days>]
```

**Options:**
- `-s, --start <date>` — New start date (required, format: YYYY-MM-DD)
- `-l, --length <days>` — New cycle length in days (optional, keeps existing length if omitted)

**Examples:**

```bash
# Move the start date back by one day
rei cycle reschedule --start 2025-01-01

# Change both start date and length
rei cycle reschedule --start 2025-01-01 --length 14
```

**Notes:**
- Only active cycles can be rescheduled
- If the cycle already has daily focus entries, the new date range must still contain all existing focus dates (otherwise the command is rejected)
- This is a correction tool — use it to fix cycles started on the wrong day, not for routine date changes

### list

List all cycles.

```bash
rei cycle list
```

**Output:**

```
Cycles (3):

  [*] 2025-01-11 - 2025-01-20 (cycle_01xyz...)
  [✓] 2025-01-01 - 2025-01-10 (cycle_01abc...)
  [x] "Failed Attempt" 2024-12-20 - 2024-12-29 (cycle_01def...)
```

Status indicators:
- `[*]` — Active
- `[✓]` — Completed
- `[x]` — Abandoned

### show

Show details of a specific cycle.

```bash
rei cycle show <cycle-id>
```

**Arguments:**
- `<cycle-id>` — Cycle ID (required)

**Output:**

```
Cycle: cycle_01abc...
  Name: Week 1
  Status: Completed
  Start: 2025-01-01
  End: 2025-01-07
  Length: 7 days
  Days with focus: 7/7
  Created: 2025-01-01 08:00

Daily Focus Selections:
  2025-01-01: Connect   5 actions  80% ✓
  2025-01-02: Study     3 actions  67% ✓
  ...

  Overall: 74% avg (5/7 adherent days)
```

The detail view includes per-day adherence data (same format as `cycle status`).

## Typical Workflow

```bash
# 1. Set up focus areas (one-time)
rei focus import anatomy-of-equanimity

# 2. Start a cycle
rei cycle start --length 10 --name "January Cycle"

# 3. Each day, select your focus
rei cycle select-focus

# 4. Check your progress
rei cycle status

# 5. At the end, complete the cycle
rei cycle complete

# 6. Start a new cycle
rei cycle start --length 10
```

## Integration with Focus Areas

Cycles work with focus areas to track daily time allocation:

- Focus areas define *what* categories of activity exist
- Cycles define *when* you're tracking (bounded time periods)
- Daily selections record *which* focus you chose each day

This creates a rhythm of intentional daily focus without rigid scheduling.

## Focus Adherence

When intentions have a focus area assigned (via `rei intention set-focus`), Rei computes **adherence** — whether your actions matched the daily focus.

- **Eligible actions**: All actions recorded that day, excluding disruption-linked actions
- **Matching**: An action matches if its intention's effective focus (own or inherited from ancestors) equals the daily focus
- **Adherent day**: A day with ≥60% matching actions
- **No-action days**: Days with zero eligible actions show `—` (not penalized)

Adherence is shown in:
- `rei cycle status` and `rei cycle show` — per-day column
- `rei today` and `rei yesterday` — daily adherence line
- `rei review day` — adherence for the reviewed day
- `rei cycle report show` — overall adherence summary

To set up focus adherence:
1. Create focus areas: `rei focus create` or `rei focus import`
2. Assign focuses to root intentions: `rei intention set-focus`
3. Start a cycle and select daily focus: `rei cycle select-focus`
4. Record actions as usual — adherence is computed automatically
