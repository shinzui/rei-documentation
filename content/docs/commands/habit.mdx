Create and manage habits. Habits are recurring practices you want to cultivate or behaviors you want to avoid.

## Habit Types

Rei supports two types of habits based on **polarity**:

- **Build habits** (default): Practices you want to cultivate and perform regularly
- **Break habits**: Behaviors you want to avoid or reduce

## Cadence Options

Habits can have different scheduling cadences:

### Fixed Schedules
- **Daily**: Every day
- **Weekly**: On a specific day of the week
- **Weekly-on**: On multiple specific days (e.g., Mon/Wed/Fri)
- **Monthly**: On a specific day of the month
- **Quarterly**: Once per quarter
- **Yearly**: On a specific date each year
- **Every N days**: Regular interval (e.g., every 3 days)

### Flexible Schedules
- **Free weekly**: At least N times per week
- **Free monthly**: At least N times per month
- **Free quarterly**: At least N times per quarter
- **Free yearly**: At least N times per year

### Cue-Based
- **Cue**: Triggered by a specific context or event (e.g., "After meals")
- **Target**: Optional frequency goal (e.g., "3/week")

## Subcommands

### create

Create a new habit.

```bash
rei habit create [options]
```

**Options:**
- `-n, --name <name>` — Habit name (required)
- `-p, --purpose <purpose>` — Why you're cultivating this habit (required)
- `-a, --action <template>` — Default action template text

**Cadence Options (pick one):**

*Fixed schedules:*
- `-d, --daily` — Daily habit
- `-w, --weekly <day>` — Weekly on a specific day (e.g., `monday`)
- `--weekly-on <days>` — Multiple days of the week (e.g., `mon,wed,fri`)
- `--monthly <day>` — Monthly on a specific day (1-31)
- `--quarterly` — Quarterly habit
- `--yearly-month <month> --yearly-day <day>` — Yearly on a specific date
- `--every <n>` — Every N days

*Flexible schedules:*
- `--free-weekly <count>` — At least N times per week
- `--free-monthly <count>` — At least N times per month
- `--free-quarterly <count>` — At least N times per quarter
- `--free-yearly <count>` — At least N times per year

*Cue-based:*
- `--cue <description>` — Triggered by a specific context/event
- `--target <frequency>` — Target frequency (e.g., `3/week`, `10/month`)

**Polarity:**
- `--break` — Create a Break habit (behavior to avoid); default is Build

**Other:**
- `--actor <actor>` — Actor performing this action (overrides global `--actor`)

**Examples:**

```bash
# Create a daily Build habit
rei habit create \
  --name "Morning Writing" \
  --purpose "Build creative fluency" \
  --daily \
  --action "Write freely for 30 minutes"

# Create a weekly habit (single day)
rei habit create \
  --name "Weekly Review" \
  --purpose "Maintain awareness of progress" \
  --weekly sunday

# Create a habit for specific days of the week
rei habit create \
  --name "Gym Sessions" \
  --purpose "Build physical strength" \
  --weekly-on mon,wed,fri

# Create a monthly habit
rei habit create \
  --name "Monthly Finance Review" \
  --purpose "Stay on top of finances" \
  --monthly 1

# Create a quarterly habit
rei habit create \
  --name "Quarterly Goals Review" \
  --purpose "Align goals with progress" \
  --quarterly

# Create a yearly habit
rei habit create \
  --name "Annual Planning" \
  --purpose "Set yearly intentions" \
  --yearly-month 1 --yearly-day 1

# Create an every-N-days habit
rei habit create \
  --name "Water Plants" \
  --purpose "Keep plants healthy" \
  --every 3

# Create a flexible weekly habit
rei habit create \
  --name "Exercise" \
  --purpose "Stay active" \
  --free-weekly 4

# Create a cue-based habit with target
rei habit create \
  --name "Mindful Breathing" \
  --purpose "Reduce stress" \
  --cue "When feeling stressed" \
  --target "3/week"

# Create a Break habit (behavior to avoid)
rei habit create \
  --name "Avoid Late-Night Snacking" \
  --purpose "Improve sleep and health" \
  --daily \
  --break
```

**Notes:**
- Use `--daily` for every day habits
- Use `--weekly <day>` for once-a-week habits
- Use `--weekly-on <days>` for multiple specific days (e.g., MWF schedule)
- Day names can be abbreviated: `mon`, `tue`, `wed`, `thu`, `fri`, `sat`, `sun`
- For yearly habits, both `--yearly-month` and `--yearly-day` are required
- Break habits support abstinence tracking (see `start-abstinence` and `log-occurrence`)

### list

List all habits.

```bash
rei habit list [options]
```

**Options:**
- `-c, --context <context>` — Filter by context
- `-w, --where <filter>` — Filter by custom property value (see examples below)
- `--json` — Output as JSON

**Examples:**

```bash
# List all habits
rei habit list

# List habits filtered by context
rei habit list --context personal

# Filter by custom property (exact match)
rei habit list --where priority=medium

# Filter by numeric property (comparison operators)
rei habit list --where "effort>=5"

# Output as JSON (for scripting)
rei habit list --json
```

**Property Filter Syntax:**
- `key=value` — Exact match (works with enum, text, and state properties)
- `key>N`, `key>=N`, `key<N`, `key<=N` — Numeric comparison (for int properties)

**Notes:**
- Linked habits inherit context from their linked intention
- Unlinked habits can have their own context set directly
- Schedule is displayed in the habit list output
- Property filters require custom properties to be defined (see [`rei custom-property`](/docs/commands/custom-property))

### show

Show details of a specific habit.

```bash
rei habit show [habit-id] [options]
```

**Arguments:**
- `[habit-id]` — Habit ID (optional if fzf is available)

**Options:**
- `--full` — Show full details including actions, focus, and linked intention
- `--json` — Output as JSON (requires `--full`)

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active habits.

**Examples:**

```bash
# Show basic habit info
rei habit show habit_01abc...

# Show full details with actions and focus
rei habit show habit_01abc... --full

# Output full details as JSON (for scripting)
rei habit show habit_01abc... --full --json
```

### record-action

Record an action for a habit with simplified workflow.

```bash
rei habit record-action [habit-id] [description]
```

**Arguments:**
- `[habit-id]` — Habit ID (optional if fzf is available)
- `[description]` — Action description (optional, uses habit's action template if not provided)

**Options:**
- `-a, --actor <actor>` — Actor that created this action (e.g., `claude-code`, `user`)
- `-t, --at <time>` — When action was performed (optional, defaults to now)
- `-d, --duration <duration>` — How long the action took (optional)

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active habits.

**Relative Time:**

The `--at` option accepts human-readable relative time expressions:
- Keywords: `now`, `today`, `yesterday`, `tomorrow`
- Natural expressions: `2 hours ago`, `30 minutes ago`, `1 day ago`
- Short forms: `-2h`, `-30m`, `-1d`, `2h`, `30m`

**Duration Formats:**

The `--duration` option accepts human-friendly duration expressions:
- Minutes: `30m`, `30min`, `30 minutes`
- Hours: `1h`, `1hr`, `2 hours`
- Combined: `1h30m`, `1h 30m`
- Decimal: `1.5h`, `0.5h`
- Bare number: `90` (interpreted as minutes)

**Examples:**

```bash
# Select habit with fzf and provide description
rei habit record-action "Completed 30 minutes of writing"

# Provide habit ID and description
rei habit record-action habit_01abc... "Morning session completed"

# Use habit's action template as description (select via fzf)
rei habit record-action

# Pipe description from stdin
echo "Writing session notes..." | rei habit record-action habit_01abc...

# Record an action that happened 2 hours ago
rei habit record-action "Morning meditation" --at "2 hours ago"

# Record yesterday's habit completion
rei habit record-action habit_01abc... -t yesterday

# Record with duration
rei habit record-action "Morning meditation" --duration 20m

# Record with both time and duration
rei habit record-action "Evening workout" --at "3 hours ago" -d 45m
```

**Notes:**
- Automatically uses the habit's linked intention if available
- Falls back to the habit's action template if no description is provided
- Simpler alternative to `action record --intention X --habit Y` when the habit is already linked to an intention

### update-action-occurred-at

Update when a habit action occurred. Useful for correcting mistakes when the wrong time was initially recorded.

```bash
rei habit update-action-occurred-at [habit-id] <time> [options]
```

**Arguments:**
- `[habit-id]` — Habit ID (optional if fzf is available)
- `<time>` — When the action occurred (required, relative time expression)

**Options:**
- `-a, --action <id>` — Action ID (optional if fzf is available)

> **FZF Support:** If IDs are not provided and [fzf](https://github.com/junegunn/fzf) is installed, interactive pickers will let you select first the habit, then from the habit's actions.

**Relative Time:**

The time argument accepts human-readable relative time expressions:
- Keywords: `now`, `today`, `yesterday`, `tomorrow`
- Natural expressions: `2 hours ago`, `30 minutes ago`, `1 day ago`
- Short forms: `-2h`, `-30m`, `-1d`, `2h`, `30m`

> **Note:** When using short forms starting with `-` (like `-2h`), use `--` to separate options from the argument.

**Examples:**

```bash
# Update with explicit IDs
rei habit update-action-occurred-at habit_01abc... --action action_01xyz "2 hours ago"

# Let fzf pick the habit and action
rei habit update-action-occurred-at "yesterday"

# Update specific habit's action to 30 minutes ago
rei habit update-action-occurred-at habit_01abc... "30 minutes ago"
```

**Notes:**
- Only works for habits that are linked to an intention
- Updates the action's occurred_at timestamp in the linked intention's event stream

## Break Habit Commands

Break habits (created with `--break`) support abstinence tracking to help you monitor how long you've avoided a behavior.

### start-abstinence

Start abstinence tracking for a Break habit. This records when you began avoiding the behavior.

```bash
rei habit start-abstinence [habit-id]
```

**Arguments:**
- `[habit-id]` — Habit ID (optional if fzf is available)

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from Break habits.

**Examples:**

```bash
# Start abstinence with explicit ID
rei habit start-abstinence habit_01abc...

# Let fzf pick the habit
rei habit start-abstinence
```

**Notes:**
- Only applicable to habits created with `--break` flag
- Cannot start abstinence if already tracking (use `log-occurrence` to reset)
- The `habit show` command will display the abstinence start date

### log-occurrence

Log an occurrence (relapse) for a Break habit. This resets the abstinence tracking.

```bash
rei habit log-occurrence [habit-id] [options]
```

**Arguments:**
- `[habit-id]` — Habit ID (optional if fzf is available)

**Options:**
- `--note <text>` — Optional note about the occurrence (context, triggers, etc.)

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from Break habits.

**Examples:**

```bash
# Log occurrence with a note
rei habit log-occurrence habit_01abc... --note "Stress triggered this"

# Log occurrence without note
rei habit log-occurrence habit_01abc...

# Let fzf pick the habit
rei habit log-occurrence --note "Late night craving"
```

**Notes:**
- Only applicable to habits created with `--break` flag
- Logging an occurrence automatically resets abstinence tracking
- Use notes to identify patterns and triggers over time

## Blockers

Habits support blockers, which represent obstacles preventing you from practicing the habit.

### Declaring a blocker

Use the `blocker` command to declare a blocker on a habit:

```bash
rei blocker declare --habit habit_01abc... "Gym closed for renovation"
```

When a blocker is declared on an active habit, the habit is **automatically paused**.

### Resolving a blocker

```bash
rei blocker resolve --habit habit_01abc... blocker_01xyz...
```

When the **last blocker** on a habit is resolved, the habit **automatically resumes**.

### Listing habit blockers

```bash
# List all habit blockers
rei blocker list --habits

# List all blockers (intentions and habits)
rei blocker list
```

See [`rei blocker`](/docs/commands/blocker) for complete blocker command documentation.

**Notes:**
- Blockers are independent from intention blockers (no propagation when linked)
- Auto-pause/resume saves you from manually managing habit status
- Blockers cannot be declared on retired habits
