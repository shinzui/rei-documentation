AI-assisted coaching and review. The agent reads your current state to provide insights and suggestions.

## Configuration

The agent requires an LLM backend. See [configuration](/docs/configuration) for setup.

## Global Options

### Debug Mode

All agent commands support `--debug` to output the rendered prompt without invoking the LLM. This is useful for testing prompt construction and debugging context loading.

```bash
rei agent --debug COMMAND
```

**Example:**

```bash
# See the prompt that would be sent for coaching
rei agent --debug coach

# See the review prompt for a specific intention
rei agent --debug review intention_01h455vb4pex

# Debug a reflection prompt
rei agent --debug reflect today
```

**Output:**
```
=== DEBUG: Rendered prompt that would be sent to agent ===

[Full prompt content with all context variables rendered]

=== END DEBUG ===
```

### Interactive Mode

All agent commands support `--interactive` (`-i`) to start an interactive Claude Code session instead of getting a one-shot response.

```bash
rei agent [-i|--interactive] COMMAND
```

**Interactive mode:**
- Starts Claude Code with your current Rei state pre-loaded
- Claude retains all tools (Bash, Read, Edit, etc.)
- All `rei` commands are pre-approved (no permission prompts)
- Supports multi-turn conversations with full context
- Claude can record actions, resolve blockers, and make changes on your behalf

**Example:**

```bash
# Interactive coaching session
rei agent coach -i

# Interactive intention review
rei agent review intention_01h455vb4pex -i
```

## Subcommands

### coach

Get coaching suggestions.

```bash
rei agent coach [options]
```

**Options:**
- `-t, --today` — Focus on today's activities

**Examples:**

```bash
# General coaching
rei agent coach

# Focus on today
rei agent coach --today

# Interactive coaching session
rei agent coach -i
rei agent coach --today -i
```

### review

Review a specific intention with AI assistance.

```bash
rei agent review [intention-id]
```

**Arguments:**
- `[intention-id]` — Intention ID (optional if fzf is available)

The agent will analyze the intention's history, actions, blockers, and related notes to provide insights and suggestions.

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active intentions.

**Examples:**

```bash
# Review specific intention
rei agent review intention_01h455vb4pex

# Let fzf pick the intention
rei agent review

# Interactive review session
rei agent review intention_01h455vb4pex -i
```

### nudge

rei agent nudge

This provides a quick suggestion based on your current intentions, blockers, and recent activity.

**Example:**

```bash
# Quick suggestion
rei agent nudge

# Interactive nudge session
rei agent nudge -i
```

**Subcommands:**

**Examples:**

### assist

Start an interactive AI assistant for setup and management of your Rei system.

```bash
rei agent assist
```

Unlike other agent commands, assist is **always interactive** — it opens a Claude Code session pre-loaded with your complete Rei state including categories, contexts, intentions, habits, blockers, and disruptions.

The assistant helps you organize, configure, and maintain your system: creating intentions, setting up categories, establishing habits, managing blockers, and keeping everything well-structured.

**Examples:**

```bash
# Start the assistant
rei agent assist

# Debug: see the prompt that would be sent
rei agent --debug assist
```

### intention-assist

Start an interactive AI assistant scoped to a specific intention and its subtree (up to 3 levels deep).

```bash
rei agent intention-assist [intention-id]
```

**Arguments:**
- `[intention-id]` — Intention ID (optional if fzf is available)

Unlike `assist`, `intention-assist` loads only the selected intention's hierarchy, its actions, blockers, memories, and guidance. Skills placed in the intention's agent directory are discovered automatically — global workspace skills are not available.

> **FZF Support:** If the ID is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from active intentions.

**Examples:**

```bash
# Start scoped assistant (FZF picker)
rei agent intention-assist

# Start scoped assistant for a specific intention
rei agent intention-assist intention_01jq8...

# Debug: see the prompt that would be sent
rei agent --debug intention-assist intention_01jq8...
```

**Intention-scoped skills** live at `<workspace>/agents/intentions/<id>/.claude/skills/<name>/SKILL.md` and are only available in `intention-assist` sessions for that intention.

### create-note

Create a note using category guidance. The agent uses the category's note guidance to structure the note appropriately.

```bash
rei agent create-note [category] -i <input> [options]
```

**Arguments:**
- `[category]` — Category slug (optional if fzf is available)

**Options:**
- `-i, --input <text>` — Input text to create note from (required)
- `--intention <id>` — Attach note to this intention
- `--habit <id>` — Attach note to this habit

> **FZF Support:** If the category is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from categories with note guidance.

**Examples:**

```bash
# Create a note in a category
rei agent create-note meeting/standup -i "Weekly standup with the team"

# Create a note attached to an intention
rei agent create-note incident/postmortem -i "Redis outage" --intention intention_01h455vb4pex

# Let FZF pick the category
rei agent create-note -i "New project kickoff"
```

**How it works:**
1. Retrieves the category's note guidance (set via `rei category set-note-guidance`)
2. Uses the LLM to generate a structured note following the guidance
3. Creates the note file in the workspace
4. Attaches it to the specified anchor (intention/habit) if provided

### update-note

Revise an existing note from natural-language instructions. Loads the current markdown body, asks the language model to produce a revised full body that incorporates the user's instructions while respecting the note's category guidance (if any), and writes the result back to the same file with a `NoteContentUpdated` event.

```bash
rei agent update-note [NOTE_ID] -I "<instructions>"
```

**Arguments:**
- `[NOTE_ID]` — Note to update (uses FZF if omitted).

**Options:**
- `-I, --instructions <text>` — Revision instructions. May also be supplied on stdin (pipe it in) instead of as an argument.

**Examples:**

```bash
# Edit a note by ID, instructions inline
rei agent update-note note_01k... -I "Add a rollback plan"

# Pick a note via FZF, instructions piped from stdin
cat instructions.md | rei agent update-note

# Inspect the prompt without invoking the LLM
rei agent --debug update-note note_01k... -I "Tighten the wording"
```

**How it works:**
1. Resolves the note ID (explicit argument or interactive FZF picker).
2. Reads the current markdown body from disk (using the note row's recorded path).
3. Looks up the note's category and reads its note guidance file (if one is set via `rei category set-note-guidance`).
4. Prompts the language model with the existing body, the category guidance, and the user's instructions, asking for the revised full body.
5. Writes the revised body to the note's file path and fires a `NoteContentUpdated` event. Downstream subscriptions (`GitCommitHandler`, `ContentProcessor`) update git, tasks, references, and links from the new body.
6. If the note is anchored to an intention, records an `Agent updated note: ...` action against that intention.

### ask-note

Ask a question about a single note, either as a one-shot answer or in an interactive Claude Code session loaded with the note as context. Unlike `update-note`, this is read-only — the note is never modified.

```bash
rei agent ask-note [NOTE_ID] -q "<question>"
rei agent ask-note [NOTE_ID] -i
```

**Arguments:**
- `[NOTE_ID]` — Note to query (uses FZF if omitted).

**Options:**
- `-q, --question <text>` — Question to ask. May also be piped on stdin instead of passed as a flag. Required for one-shot mode; ignored in interactive mode.
- `-i, --interactive` — Start an interactive Claude Code session with the note pre-loaded as context.

**Examples:**

```bash
# One-shot question via flag
rei agent ask-note note_01k... -q "What does this note imply for next week?"

# One-shot question via stdin
echo "Summarise this in three bullets" | rei agent ask-note note_01k...

# FZF picker + question via flag
rei agent ask-note -q "What's the next action here?"

# Interactive chat about a note
rei agent ask-note note_01k... -i

# Inspect the rendered prompt without invoking the LLM
rei agent --debug ask-note note_01k... -q "What's the gist?"
```

**How it works:**
1. Resolves the note ID (explicit argument or interactive FZF picker).
2. Reads the current markdown body from disk (using the note row's recorded path).
3. **One-shot mode:** substitutes the body and the question into an embedded prompt template, calls `claude --print`, prints the answer, and records an `AgentSession` with focus type `ask_note` and the note id as the focus target.
4. **Interactive mode (`-i`):** prepends the note (id, title, path, full body) to the standard Rei system context and launches Claude Code as a child process. Records an interactive session with focus type `ask_note` before launching. The user chats freely in the TUI; on exit, the shell prompt returns.

In both modes the note file is read-only — the command never writes to it. To edit a note, use `rei agent update-note` instead.

### explore

Explore a collection's contents in an interactive Claude Code session pre-loaded with the collection's notes. Ask follow-up questions, drill into individual notes, or have Claude record actions on your behalf — all with the collection's context already loaded.

```bash
rei agent explore [COLLECTION] [options]
```

**Arguments:**
- `[COLLECTION]` — Collection name or ID (optional if fzf is available)

**Options:**
- `-n, --limit N` — Maximum number of notes with content (default 20)
- `--no-interactive` — Run one-shot instead of starting an interactive session (useful for scripting and piping)
- `--debug` — Show the prompt without calling the agent

> **FZF Support:** If the collection is not provided and [fzf](https://github.com/junegunn/fzf) is installed, an interactive picker will let you select from existing collections.

**Examples:**

```bash
# Explore a collection by name (interactive by default)
rei agent explore "Research Papers"

# Let FZF pick the collection
rei agent explore

# One-shot summary instead of interactive session
rei agent explore "research" --no-interactive

# Limit notes for large collections
rei agent explore "Category Notes" --limit 5

# Debug: see the prompt
rei agent --debug explore "Category Notes"
```

See [`rei collection`](/docs/commands/collection) for full collection management commands.

**Subcommands:** `create`, `list`, `show`, `pause`, `resume`, `complete`, `fire`, `update-guidance`

## Modes Comparison

| Mode | Behavior | Use Case |
|------|----------|----------|
| Non-interactive (default for most commands) | Returns a single response, read-only | Quick advice, suggestions |
| Interactive (`-i`) | Multi-turn Claude Code session | Getting unstuck, making changes |

Some subcommands always run interactively (`assist`, `intention-assist`) and have no flag. `explore` defaults to interactive — pass `--no-interactive` to get a one-shot response instead.

## Important Notes

- **Non-interactive mode** is read-only — the agent does not modify your data
- **Interactive mode** gives Claude full CLI access — it can record actions, resolve blockers, etc.
- All changes in interactive mode are attributed with `--actor claude-code`
- The agent reads your events, markdown files, and read models to understand your current state
