Rei is built on a simple conviction: **the right foundation makes everything, including AI agents, more effective.** That foundation is intentionality: organizing life by meaning, recording evidence of change, and reflecting deliberately on what happened.

Agents amplify whatever system they're built on. If the system is a task list, agents become task-completion machines. If the system is built around intention, evidence, and reflection, agents become thinking partners that understand *why* you're doing what you're doing.

## The Nine Primitives

Everything in rei is built from nine composable primitives. These represent the minimum set of concepts needed to capture intentional action over time.

### Intention & Execution

| Primitive | What it captures |
|-----------|-----------------|
| **Intention** | A commitment to a meaningful change in the world. Not a task, but a promise with identity. |
| **Action** | The concrete work you (or an agent) do. Attributed, timestamped, immutable. |
| **Outcome** | Observable evidence that something changed. Proof, not self-report. |
| **Blocker** | What prevents the next meaningful step. Makes obstacles visible and actionable. |
| **Dependency** | Another intention that constrains this one. Makes relationships explicit. |

### Practice & Reflection

| Primitive | What it captures |
|-----------|-----------------|
| **Habit** | A recurring activity with lifecycle management. Build habits, break habits, track consistency. |
| **Reflection** | Deliberate interpretation of past work. What happened, what it means, what to do next. |

### Time & Focus

| Primitive | What it captures |
|-----------|-----------------|
| **Focus** | A category of activity for daily time allocation. See where your time goes. |
| **Cycle** | A bounded time period for tracking focus selections. Rhythm, not rigidity. |

## The Agent Layer

On top of these primitives, rei adds a full agent system that operates within the same event-sourced model:

| Concept | What it enables |
|---------|----------------|
| **Agent Sessions** | Every agent interaction is recorded with timing, tokens, model, and summary |
| **Actor Identity** | Every event is attributed to human or agent. You always know who did what. |
| **Delegation** | Assign intentions to agents with explicit scope (research, draft, execute) and autonomy level |
| **Review Checkpoints** | Agents submit work for your approval. Structured human-in-the-loop oversight. |
| **Agent Memory** | Agents build persistent understanding across sessions: facts, patterns, preferences, constraints |
| **Scheduled Automation** | Agents run on recurring schedules or react to events. Background work that's always traceable. |

## Shared Structured Context

Custom properties and state machines create a communication layer between you and your agents:

| Concept | What it enables |
|---------|----------------|
| **Custom Properties** | Typed metadata (paths, URLs, quantities, booleans) that agents read for context when coaching or executing work |
| **State Machines** | Full workflow discipline with enforced transitions, WIP limits, checklist-gated gates, and lifecycle automation |
| **Category Scoping** | Properties that vary by domain. Your work workflow differs from your personal workflow. |

When you set a `project_path` on an intention, the agent knows where to look. When an intention enters a `review` state in your workflow, the agent understands what phase it's in. Properties turn implicit context into explicit, queryable, agent-readable facts.

The primitives provide the *meaning*. The agents provide the *capacity*. Custom properties provide the *shared context*. The event log provides the *truth*.

## Five Principles

### 1. Intention-Centered

Organize work by meaning, not checklists. An intention is not a task; it's a commitment to change something in the world. Tasks are disposable; intentions have identity. The hierarchy of intentions connects daily work to deeper purpose.

### 2. Agent-Augmented

AI agents coach, assist, execute, and automate, always within boundaries you define. From read-only coaching to fully delegated work, you control the autonomy level. Agent actions are recorded alongside your own in the same timeline.

### 3. Event-Sourced Truth

Every change is recorded as an immutable event. State is always derived, never mutated. This means complete history, no data loss, full auditability, and the ability to rebuild any view from events. Human and agent work share the same event model.

### 4. Evidence-Based Review

Outcomes and reflections beat memory. Instead of checking boxes, you record evidence of progress and reflect on what it means. Agents assist the reflection process by surfacing patterns and asking questions.

### 5. Local-First & Transparent

Your files are yours; nothing is hidden. The workspace is a folder of markdown files. The database stores events, but knowledge lives in files you control. Agent work is fully visible: sessions, actions, delegations, reviews.
